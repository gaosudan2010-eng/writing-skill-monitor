# 文本创作 Skill 分类审计

审计日期：2026-07-21

## 审计范围

- 逐条检查 360 个原子 Skill 的名称、说明、标签和主要交付物。
- 检查 18 个合集 / 项目是否被误计为原子 Skill。
- 对 21 个一级类别执行类别内一致性与跨类别边界检查。

## 分类优先级

1. **对象层级优先**：包含多个可独立安装 Skill 的仓库根对象归入“合集 / 项目”。
2. **专属交付对象优先**：只服务论文、小说、社媒等特定对象的能力，优先归入对应对象类别。
3. **主任务优先**：同一 Skill 同时包含生成、润色和发布时，按用户使用它要完成的核心任务分类。
4. **通用动作兜底**：跨对象的校对、改写、去 AI 和清晰度优化归入“编辑润色”。
5. **关键词只作证据**：平台名、格式名或行业词偶然出现在说明中，不单独触发改类。

## 本轮主要修正

- 小说分析、扫描、审稿和小说全流程能力从“编辑润色”迁入“小说与故事”。
- 学术去 AI、Nature 润色和顶会论文润色从“编辑润色”迁入“学术论文”。
- 社媒帖子评分和通用社媒资料优化能力迁入“社交媒体”。
- 市场报告、行业研究报告迁入“调研写作”。
- 写作声音提取能力迁入“品牌与营销”。
- 面向营销文案逻辑、说服力和转化表达的 `copy-editing` 迁入“品牌与营销”；通用校对仍保留在“编辑润色”。
- 9 个经仓库目录核验的多 Skill 项目从原子 Skill 迁入“合集 / 项目”。

## 保留的边界案例

- `Email Formatter` 归“邮件与销售”：虽然执行润色，但主要交付物是可发送邮件。
- `linkedin-profile-optimizer` 归“简历与求职”：其目标是职位搜索和求职转化，而非日常社媒运营。
- `wechat-auto-publisher` 归“创作工具”：核心增量是草稿管理和自动发布，不是单篇公众号成稿。
- `去 AI 味内容引擎` 归“编辑润色”：尽管面向中文媒体平台，核心任务仍是自然化改写和去模板腔。
- `nature-paper-to-patent` 归“产品与技术”：交付物是技术方案的专利交底；当前样本不足以单设“法律与知识产权”。
- `contract-and-proposal-writer` 归“企业写作”：其主要使用场景是自由职业项目合同、提案和工作说明书。

## 后续维护规则

- 新增 Skill 先判断对象层级，再判断交付对象，最后判断执行动作。
- 仅对高置信度误分使用名称覆盖；通用规则必须限制适用的原类别，避免跨域误伤。
- 当某个边界类别出现至少 5 个用途一致、交付物明确的原子 Skill 时，再评估是否新增一级分类。
